# Contributing

**This project does not accept external code contributions at this
time.** Please do not open pull requests. Unsolicited pull requests
will be closed without review and without merging, regardless of
quality.

This is not a judgment about would-be contributors or their work. It
is a deliberate legal/administrative position: this repository is
maintained by a single copyright holder, and that is what makes the
dual-licensing arrangement described in `README.md` (AGPL-3.0-or-later,
with a separate commercial licence available on request) possible.
Accepting a third-party pull request would introduce a second
copyright holder into the codebase, and relicensing the affected code
would then require that contributor's separate consent. To keep the
licensing position simple and unambiguous, the project is kept to a
single author.

This policy may change in the future. If it does, this file will be
updated first, and any new contribution process will be described
here before it opens.

## Issues

Bug reports and questions are welcome as GitHub issues. Opening an
issue does not imply any obligation to act on it, and issues proposing
a specific code change may still be declined for the same reason as
above.

## Security

If you believe you have found a security-relevant defect, please
report it privately (see `README.md` for contact details) rather than
opening a public issue.
