# Third-Party Notice: Stone Soup

This repository's tracking adapter (step 10 of the implementation order
in `docs/design/TRACKING_EVIDENCE_TO_RATIONAL_ADAPTER_SPEC.md`)
optionally uses **Stone Soup** as a deterministic tracking and
scenario-generation layer that emits evidence into the adapter's
existing `tracking-adapter/v1` boundary. Stone Soup is not a dependency
of the exact adapter (`tracking_adapter_format.py`, `tracking_adapter_
canon.py`, `tracking_adapter_verifier.py`, `tracking_adapter_
certificate.py`) or of R21 itself — see `tests/test_stonesoup_import_
boundary.py`, which proves this mechanically.

```text
Source repo:  https://github.com/dstl/Stone-Soup
```

Install via `requirements-stonesoup.txt` (this repository's own file,
generated by `pip-compile --generate-hashes`, pinning Stone Soup's
complete transitive dependency tree — matplotlib, numpy, scipy, and
Stone Soup's other plotting/estimation dependencies — by hash, not just
by version):

```sh
pip install -r requirements-stonesoup.txt
```
